# Estimation

Complexity dimensions, each scored 0–5:

- breadth;
- coupling;
- domain logic;
- contract and data;
- testing difficulty;
- operational complexity;
- uncertainty.

Keep risk separate from complexity.

Effort output:

- optimistic hours;
- likely hours;
- pessimistic hours;
- PERT expected hours: `(O + 4M + P) / 6`;
- confidence and assumptions.

Estimates must include implementation, testing, review/repair and coordination where applicable.
